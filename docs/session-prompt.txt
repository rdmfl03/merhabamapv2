## Database Contract (CRITICAL)

The current database schema is a PROTECTED CONTRACT.

Reason:
- `merhabamap-ingest` may depend on current schema structure
- breaking it can destroy ingest compatibility and production stability

You MUST:

- first understand existing DB schema (tables, relations, constraints)
- assume schema is intentionally stable
- prefer application-layer solutions over schema changes

You must NEVER:

- rename tables
- drop tables
- rename columns
- remove columns
- repurpose fields
- change relation meanings
- introduce breaking migrations
- change schema just to support a feature

If a change seems to require DB modification:

1. explain WHY it is needed
2. explain COMPATIBILITY RISK (especially ingest)
3. propose NON-SCHEMA alternative
4. WAIT for explicit approval
